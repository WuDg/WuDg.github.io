Ngrok实现内网穿透

内网穿透的作用👀

  • 把本地的服务映射到外网服务，即通过IP直接访问本地的服务
  • 把本地磁盘当做文件管理服务器，如视频、图片等，通过内网穿透，可通过外网访问到本地磁盘文件

如何搭建内网穿透服务🙌

在服务器上拉取 github 项目

一台云服务器,一个域名,并且域名泛解析解析到云服务器(即在域名解析添加如下记录)

安装环境(安装gcc和git)🤷‍♂️

安装go语言环境🎉

检查环境安装🐱‍🚀

在服务器上搭建Ngrok服务😉

下载ngrok源码
生成证书🎶
将下面内容写入到shell脚本中(如新建crt.sh)，并赋予脚本执行权限 chmod u+x crt.sh😢

执行命令 ./crt.sh 参数1 参数2🌹

  • 参数1:你的域名，如 abc.com
  • 参数2:你的ngrok根路径,如 /usr/local/ngrok

Linux 平台 32 位系统：GOOS=linux GOARCH=386
Linux 平台 64 位系统：GOOS=linux GOARCH=amd64
Windows 平台 32 位系统：GOOS=windows GOARCH=386
Windows 平台 64 位系统：GOOS=windows GOARCH=amd64
MAC 平台 32 位系统：GOOS=darwin GOARCH=386
MAC 平台 64 位系统：GOOS=darwin GOARCH=amd64
ARM 平台：GOOS=linux GOARCH=arm

进入bin目录下执行下面命令编译生成linux64位版ngrok服务端🐱‍👤

生成在~/ngrok/bin/目录中

编译生成windows64位版ngrok客户端👨

生成在~/ngrok/bin/windows_amd64/目录中

客户端同目录下新建ngrok.cfg配置文件，以下为其中的内容🎁
服务器后台8087端口启动ngrok服务,注意其中的域名换成你自己的🤦‍♂️
  • -httpAddr=80 此配置用来指定映射域名http访问的端口 默认80
  • -httpsAddr=443 https的访问端口 默认443
  • -tunnelAddr=4443 客户端连接的端口 默认4443

注意打开服务器对应的ngrok端口,可以配置为80端口则方便通过域名直接访问本地服务，但是由于线上的大多数同时也运行着nginx，会产生冲突，这里可以把ngrok端口设置为非80端口，再使用nginx代理一下即可

ngrok客户端同目录新建start.bat文件(前面配置了泛域名，此处的abc任取)✔
控制台打印一下内容则表示成功(即把本地端口为8086的服务映射到云服务器的8087端口上)👍
